## 文档标准

每份案例文档必须包含以下组件：

| 组件 | 必需 | 说明 |
|------|------|------|
| TL;DR | ✓ | 主论断 + 边界条件句 |
| 逻辑审计表 | ✓ | 7 列：命题/逻辑结构/反驳谬误/证据/来源权重/信心/证伪条件 |
| 推导链 | ✓ | 箭头骨架，不塞表格 |
| 失效路径 | ✓ | 3 步，含时间阈值 |
| SORRY | ✓ | 未决实证缺口 |
| CALIBRATE | ✓ | 操作化模糊点 |
| 对抗测试 | ✓ | 4 维：结构/极端/反向/历史失败 |
| STATUS_REPORT | ✓ | 轮次/记忆/SORRY 数/格式纪律自评 |

## 引用规则

**单向引用**：
- ✅ 核心层（Fc v9.x）**可引用**本池的实证锚点与操作化细节
- ❌ 本池**不可反向引用**核心层作为证明（避免循环论证）
- ✅ 本池之间**可交叉引用**同类议题的实证数据

**来源权重分级**：
- **S**：逻辑/物种事实（公理层）
- **A**：一手官方数据/司法案例/学术研究
- **B**：跨国代理/商业报告/政策文本
- **C**：估算值/待收集/媒体引用

## 版本控制

**命名规则**：`v{主版本}.{次版本}-DRAFT.md`

- 首次生成：`v1.0-DRAFT`
- 格式修正：`v1.1-DRAFT`
- 实证补全：`v1.2-DRAFT`
- 审计通过：`v2.0-DRAFT`（仍不标记 HARDENED）

**状态锁定**：
- 未决 SORRY > 0 → **强制 DRAFT**
- 未决 CALIBRATE > 3 → **建议暂缓引用**
- 真实盲区（不可搜索）→ **标记为 CALIBRATE-不可达**

## 层间绝缘

本池与 Fc 核心层的关系：

```
Fc 核心层（v9.x HARDENED）
    ↓ 单向引用实证锚点
Fc 应用案例池（fc-applications DRAFT）
    ↓ 提供操作化案例
外部数据源（国家统计局/学术数据库/司法案例）
```

**禁止**：将本池的任何推导结论反向注入核心层公理。
